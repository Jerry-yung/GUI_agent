## 3. 每轮循环的输入接口（Agent 每轮必须等价拿到）

Multi-path 是 `while round_count < MAX_ROUNDS` 循环（默认 20 轮，`appagent_multipath.py:136`）。每轮 Agent 的输入必须包含以下 5 项：

### 3.1 截图（带 c/s 标注，强制 1080×2400）
**官方生成链路**（`appagent_multipath.py:158-174`）：
```python
click_actions, input_actions_pre, scroll_actions, current_page_all_actions = actions_generate(html_content, xml_content)
click_actions, input_actions, scroll_actions, scroll_action_bounds = current_actions_generate(
    current_action_infos, click_actions, input_actions_pre, scroll_actions, current_page_all_actions
)

drawn_screenshot = os.path.join(task_dir, current_page_name, current_page_name + '_labeled_multi.png')
draw_bbox_multi(screenshot_path, drawn_screenshot, click_actions)          # 标 c1, c2...
draw_bbox_multi(drawn_screenshot, drawn_screenshot, scroll_action_bounds)   # 标 s1, s2...

imgcv = cv2.imread(drawn_screenshot)
imgcv = cv2.resize(imgcv, (1080, 2400))   # ← 强制缩放
cv2.imwrite(drawn_screenshot, imgcv)
```

**你的 Agent 必须**：拿到 `{page}_labeled_multi.png`（1080×2400），或你自己调用 `utils.actions_generate()` + `utils.draw_bbox_multi()` + `cv2.resize(1080, 2400)` 生成等价的标注图。

**⚠️ 关键**：`c1/c2...` 和 `s1/s2...` 的编号顺序由 `actions_generate()` → `current_actions_generate()` 的输出顺序决定。如果你自己生成候选动作，编号体系必须与官方一致，否则 Agent 输出的 `click(c5)` 会指向错误元素。

### 3.2 总任务描述（`task_desc`）
来源：`task["task"].split('\n')[-1]`（"简洁任务"行）。

### 3.3 当前子步骤描述（`current_task_desc`）
代码（`appagent_multipath.py:179-183`）：
```python
task_count = current_page_name.count('_')
if task_count < len(multi_task_desc) - 1:
    prompt = re.sub(r"<current_task_desc>", multi_task_desc[task_count], prompt)
else:
    prompt = re.sub(r"<current_task_desc>", multi_task_desc[len(multi_task_desc) - 1], prompt)
```

含义：第 1 步（`current_page_name = QQmusic0`，`task_count = 0`）用 `multi_task_desc[0]`（即 "1. xxx"），第 2 步（`QQmusic0_34`，`task_count = 1`）用 `multi_task_desc[1]`，以此类推。若步数超过分步条数，重复用最后一条。

### 3.4 历史摘要（`last_act`）
代码（`appagent_multipath.py:228`）：`last_act = res[-1]`，即模型上一轮输出的 `Summary` 字段。

### 3.5 结构文本（可选）
如果你的 Agent 需要 HTML/XML 做辅助理解，必须读取同目录下的 `-html.txt` 和 `-xml.txt`。

## 4. Agent 输出接口（每轮必须能被解析）

官方使用 `parse_multi_explore_rsp()`（`appagent/model.py:591`）解析模型响应。

### 4.1 输出格式要求
模型响应必须包含以下四行（`re.MULTILINE` + `$` 匹配，**不支持多行换行**）：
```
Observation: ...
Thought: ...
Action: click(c5)          # 或 scroll(s2, "up") / input("hello") / back("back")
Summary: ...
```

**⚠️ 关键差异**：Multi-path 使用 `parse_multi_explore_rsp`，不是 `parse_explore_rsp`。后者用 `re.DOTALL` 支持多行，前者用 `re.MULTILINE` 且以 `$` 结尾，意味着 `Action:` 行必须在一行内结束，不能换行后再写参数。

### 4.2 四种合法 Action

| 动作 | Action 行示例 | 解析后返回 |
|------|--------------|-----------|
| click | `Action: click(c5)` | `["click", "c5", last_act]` |
| scroll | `Action: scroll(s2, "up")` | `["scroll", "s2", "up", last_act]` |
| input | `Action: input("hello")` | `["input", "hello", last_act]` |
| back | `Action: back("back")` | `["back", last_act]` |

**方向约束**：scroll 的 direction 必须是 `"up"`、`"down"`、`"left"`、`"right"`（含双引号）。

## 5. 动作映射与页面跳转接口（Multi-path 核心硬约束）

这是 Multi-path 与 Single-path 最大的不同：Agent 的动作**直接决定下一页**，而不是跟随固定 GT 路径。

### 5.1 动作过滤（Multi-path 特有，Single-path 没有）
代码（`appagent_multipath.py:153-163`）：
```python
current_page_all_action_ids = current_page_actions.get(current_page_name, [])
current_action_infos = []
for action_id in current_page_all_action_ids:
    current_action_infos.append(id_to_action[int(action_id)])

click_actions, input_actions_pre, scroll_actions, current_page_all_actions = actions_generate(html_content, xml_content)
click_actions, input_actions, scroll_actions, scroll_action_bounds = current_actions_generate(
    current_action_infos, click_actions, input_actions_pre, scroll_actions, current_page_all_actions
)
```

**含义**：
- `current_page_actions` 来自 `all_page_actions.json`，是当前页**合法**的动作 id 列表。
- `current_action_infos` 是这些 id 对应的**完整动作字符串**列表（如 `["click(<p id='...'>歌曲队列</p>)", "scroll(([360,801],[360,267]))"]`）。
- `current_actions_generate()` 把 `actions_generate()` 生成的全部候选动作，按 `current_action_infos` **过滤**，只保留当前页合法的动作。

**你的 Agent 必须**：基于过滤后的 `click_actions` / `scroll_action_bounds` / `input_actions` 做选择。如果 Agent 选了一个被过滤掉的编号，后续 `action_click()` 会因 `area_idx` 越界或查不到 `all_action_ids` 而返回 `ERROR`。

### 5.2 Click 动作映射链路
代码（`appagent_multipath.py:231-244`）：
```python
area_idx = int(re.findall(r'(\d+)', str(area))[0])
current_page_name, action_info, action_id = action_click(
    click_actions, area_idx, current_page_all_actions,
    all_action_ids, current_page_name, all_page_convert,
)
```

`utils.action_click()` 内部（`utils.py:457-467`）：
```python
def action_click(click_action, area_idx, actions_list, all_action_ids, current_page_name, all_page_convert):
    action_info = actions_list[click_action[area_idx - 1]]   # actions_list = current_page_all_actions
    click_action = 'click(' + action_info + ')'
    action_id = all_action_ids[click_action]
    new_page_name = all_page_convert[current_page_name + 'act' + str(action_id)]
    return new_page_name, click_action, action_id
```

**完整数据流**：
```
Agent 输出 click(c5)
    ↓
click_actions[5-1] = "click(歌曲队列, [360,801][360,267])"   # 简化格式，带 bounds
    ↓
current_page_all_actions["click(歌曲队列, [360,801][360,267])"] = "<p id='...'>歌曲队列</p>"   # 完整 HTML
    ↓
click_action = "click(<p id='...'>歌曲队列</p>)"   # 拼成 all_action_id 的 key
    ↓
all_action_ids["click(<p id='...'>歌曲队列</p>)"] = 34   # 全局 action_id
    ↓
all_page_convert["QQmusic0act34"] = "QQmusic0_34"   # 查三元组得到下一页
    ↓
return "QQmusic0_34", "click(<p id='...'>歌曲队列</p>)", 34
```

**⚠️ 致命约束**：`all_action_id.json` 的 key 是**完整 HTML 字符串**（含标签、属性、空格）。你的 Agent 如果只输出 `click(歌曲队列)` 或坐标，必须通过 `current_page_all_actions` 映射表查到完整 HTML，才能拿到正确的 `action_id`。任何一步字符串不匹配都会导致 `action_id` 错误或为 `-2`。

### 5.3 Scroll 动作映射链路
代码（`appagent_multipath.py:245-258`）：
```python
area_idx = int(re.findall(r'(\d+)', str(area))[0])
current_page_name, action_info, action_id = action_scroll(
    scroll_action_bounds, area_idx, direction, all_action_ids,
    current_page_name, all_page_convert,
)
```

`utils.action_scroll()` 内部（`utils.py:470-478`）：
```python
def action_scroll(scroll_action_bounds, area_idx, direction, all_action_ids, current_page_name, all_page_convert):
    scroll_action = dict_scroll_parameters(scroll_action_bounds[area_idx - 1], direction)
    scroll_action = "scroll(" + str((scroll_action)) +")"
    action_id = all_action_ids[scroll_action]
    new_page_name = all_page_convert[current_page_name + 'act' + str(action_id)]
    return new_page_name, scroll_action, action_id
```

**关键**：`dict_scroll_parameters()` 把 bounds 和方向转换为坐标对，然后 `str((scroll_action))` 会生成类似 `scroll(([360, 801], [360, 267]))` 的字符串。这个字符串必须**精确匹配** `all_action_id.json` 中的 key。你不能自己计算坐标，必须用官方的 `dict_scroll_parameters()`。

### 5.4 Input 动作映射链路
代码（`appagent_multipath.py:259-269`）：
```python
current_page_name, action_info, action_id = action_input(
    text, input_actions, current_page_all_actions, all_action_ids,
    current_page_name, all_page_convert,
)
```

`utils.action_input()` 内部（`utils.py:481-500`）：
- 遍历 `input_actions`（格式如 `input(带有文本xxx的输入框, [bounds])`）
- 用正则 `r",\s*'([^']+)'"` 提取输入框文本
- `comput_f1(text, result) >= 0.5` 则匹配成功
- 匹配到的动作字符串直接查 `all_action_ids`

**约束**：Agent 输入的文本与候选输入框文本的 F1 分数必须 ≥ 0.5，否则会返回 `ERROR`（`action_id = -2`）。

### 5.5 Back 动作
代码（`appagent_multipath.py:272-275`）：
```python
elif act_name == "back":
    current_page_name = current_page_name.rsplit('_', 1)[0]
    ans_action_id.append(-1)
    ans_action_info.append('back')
```

**含义**：直接回退到上一级页面（去掉最后一个 `_` 后缀），不查 `all_page_convert`。

## 6. 页面推进逻辑（Multi-path 与 Single-path 的本质区别）

| 维度 | Multi-path（本文档聚焦） |
|------|------------------------|
| 当前页来源 | 上一步动作通过 `all_page_convert` 动态决定 |
| 截图加载 | 基于 `current_page_name`（Agent 实际所在页） |
| 动作过滤 | **必须**用 `current_actions_generate()` 过滤合法动作 |
| 结束条件 | `current_page_name == final_page_name` 或达到 `MAX_ROUNDS` |
| 结果 `page_name` | 写入 Agent **实际到达**的页面名（`new_page_name`） |

**⚠️ 致命陷阱**：Multi-path 的结果文件中 `page_name` 是 Agent 实际到达的页面，不是 GT 路径名。如果你把 Single-path 的结果（写 GT 页名）拿给 `multi_path_test` 评测，会得出完全错误的结论。

## 8. 评测接口（调用 `eval.py`）

### 8.1 评测函数
```python
from eval import multi_path_test
multi_path_test('results/multi_simpleqwen2.5-vl', 'simple')   # simple 任务
# 或
multi_path_test('results/multi_complexqwen2.5-vl', 'complex')  # complex 任务
```

### 8.2 评测逻辑（`eval.py:120-178` 确认）

```python
file_name = file.split('.')[0]          # "QQmusic0_34_347_0"
real_path = file_name.split('_')[1:-1]   # ['34', '347']（GT 路径后缀）

# 读取结果文件，在特定行数截断
if task_type == 'complex':
    if i == 24 or 到达目标页: break
else:
    if i == 19 or 到达目标页: break

# 只取最后一行的 page_name
pred_path = pred_paths[-1].split(':')[-1].strip().split('_')[1:]  # 如 ['34', '347']

# 对比 pred_path 与 real_path 的逐段后缀
true_action_sum = 0
for i, path in enumerate(pred_path):
    if path == real_path[i]:
        true_action_sum += 1
```

**关键发现**：
- `multi_path_test` 的 `success_rate` 检查的是：**Agent 最终到达的页面路径是否包含完整 GT 路径作为前缀**（`true_action_sum == len(real_path)`）。
- `action_acc`（第一个值）= `true_action_sum / gt_action_sum`，即路径后缀匹配数除以 GT 步数。
- 评测**不直接对比 action_id**，但 `action_id` 决定了 Agent 能否通过 `all_page_convert` 正确跳转，因此间接决定 `page_name` 是否正确。
- `complex` 任务最多读 24 行，`simple` 最多读 19 行（硬编码）。

## 10. 检查清单（跑通前逐项确认）

### 数据准备
- [ ] `convert_mobile3m.py --mode explore` 已对 `QQmusic_graph_8152` 执行

- [ ] 任务 JSON（如 `simple_tasks_sample.json`）与图目录在同一 `data_dir` 下

### Agent 输入
- [ ] 每轮截图是 `{page}_labeled_multi.png`（1080×2400，带 c/s 标注）
- [ ] Prompt 包含总任务描述 + 当前子步骤 + 历史摘要（与官方 `multipath_task_template` 等价）
- [ ] 若 Agent 需要结构文本，提供的是同目录下的 `-html.txt` / `-xml.txt`

### Agent 输出
- [ ] 输出可被解析为 `click(cN)` / `scroll(sN, "dir")` / `input("text")` / `back("back")`
- [ ] 使用 `parse_multi_explore_rsp` 兼容格式（四段，Action 行内结束）
- [ ] 或：自写解析器，但解析后的 `area_idx`、`direction`、`text` 必须与原逻辑一致

### 动作映射
- [ ] Click：通过 `click_actions[area_idx-1]` → `current_page_all_actions` → 完整 HTML → `all_action_ids` 查 `action_id`
- [ ] Scroll：通过 `scroll_action_bounds[area_idx-1]` + `direction` → `dict_scroll_parameters` → `all_action_ids` 查 `action_id`
- [ ] Input：通过 F1 ≥ 0.5 匹配 `input_actions` 中的候选 → `all_action_ids` 查 `action_id`
- [ ] 所有查表用的 key 字符串与 `all_action_id.json` 中的 key **完全一致**

### 页面跳转
- [ ] Multi-path：使用 `action_click` / `action_scroll` / `action_input` 返回的 `new_page_name` 作为下一轮 `current_page_name`
- [ ] 结果文件中 `page_name` 写的是 Agent **实际到达**的页面名

### 结果文件
- [ ] 路径：`{save_path}/{task_type}{model_name}/{task_name}_{index}.txt`
- [ ] 每行：`{action_id}: {action_info}:{page_name}`（注意第二个冒号后无空格）
- [ ] 失败时 `action_id` 写 `-2`（映射/解析失败）或 `-3`（未知动作类型）

### 评测
- [ ] 调用 `multi_path_test(result_dir, 'simple')` 或 `multi_path_test(result_dir, 'complex')`
- [ ] 索引模式与推理结果一致（均为 explore 模式）

---

## 11. 一句话总结

> 在 Multi-path QQmusic 测试中，你的 Agent 必须：**拿到官方标注图 → 输出 `click(cN)`/`scroll(sN,dir)`/`input("text")`/`back` 格式 → 通过官方 `current_actions_generate` 过滤合法动作 → 经 `action_click`/`action_scroll`/`action_input` 映射为全局 `action_id` 并动态跳转到下一页 → 把 `action_id: action_info:实际到达页` 写入官方路径的结果文件 → 由 `multi_path_test` 评测最终页面路径与 GT 路径的后缀匹配度。**

其中 **`all_action_id.json` 的完整 HTML key 精确匹配** 和 **Multi-path 的动态页面跳转** 是两个最容易导致评测失败的硬约束。
